层次缩进

缩进方法

层次缩进其实很简单, 就是被不同层的{...}的代码前面要用tab缩进. 用到的花括号的地方有很多, 比如if, for, while, method {...}等等.

这里举个例子:

- (void)thisIsAMethodWithAParam:(NSObject *)aParam {
....if (a bool value) {
........for (int i = 0; i < 5; i++) {
............while (a bool value) {
................do something.
............}
........}
....}
}
public class AAA {

}

就这样在每一个花括号中的代码都用相同个数的tab缩进, 使得代码拥有层次感, 增加易读性. 对于不同层次的花括号用不同长度的tab来区分, 相邻层次的用1个tab来区分.

而 Xcode 是支持代码收缩的, 开启代码收缩的方法是

  1. Xcode -> Preferences...
  2. Text Editing
  3. 打开 Code folding ribbon
  4. 打开 Focus code blocks on hover

之后, 就可以在 Xcode 的编辑器中的左侧, 根据不同灰度的缩进提示来收缩代码快.

另外, 介绍一个用于快速收缩所有方法的快捷键

option + command + shift + ←

和一个快速展开所有方法的快捷键

option + command + shift + →

收缩你的方法可以让现在不必要看到的方法尽可能的避免占用你的视线.

关于花括号

在这里说明两种花括号的使用方法.

Java:
public class Tarena {
    public static void main () {
        code...
    }
}

C:
void function()
{
    code...
}

Java 风格的花括号喜欢把 { 直接跟在后面.

C 风格的花括号喜欢另起一行写 {.

在 OC 中, 没有任何硬性规定用哪种写法, 你可以随意选择.

但是我比较喜欢 Java 风格的写法, 这样如果收缩代码后, 代码只占有一行, 并且不影响用视线遍历方法的返回值类型.

空格分割

空格是一个让代码美观的必要工具. 在 OC 中没有关于空格的规定, 但是在 Swift 中, 空格就变得很有必要. 所以为了让代码美观, 为了学习 Swift, 习惯在某些地方打空格是非常必要的.

下面举几个栗子(下划线代表空格):

@property_(nonatomic,_strong)_NSString_*name;

-_(void)aMethodWithString:(NSString_*)string_andArray:(NSArray_*)array_{
____NSMutableArray_*newArray_=_[NSMutableArray_array];
____for_(int_i_=_0;_i_<_array.count;_i++)_{
________[newArray_addObject:array[i]];
____}
}

-_(void)testMethod_{
    [self_aMethodWithString:@"aaa"_andArray:@[@"aaa",_@"bbb"]];
}

你们细细体会, 总结几点就是

  1. ,后面跟空格
  2. =左右两边有空格, +, -, *, /, >, <同理
  3. 数据类型和变量之间有空格
  4. 方法前缀-和+后有空格(这一点很多人都会忽略)

方法排序

方法排序很有讲究, 如果一个文件(尤其是UIViewController)中方法有很多, 那么方法的排序就会影响到这个文件的易读性.

方法的排序你可以遵循以下几点

  1. 重写(父类)的方法写在前面
  2. 经常去修改和检查的方法写在前面, 很少修改和检查的方法写后面
  3. 公开的方法写前面, 不公开的方法写后面
  4. 同一个协议方法写在一起, 所有的协议方法依次排列

通过以上的赛选和排序, 总结起来一个文件结构可以如下

#pragma mark - single object
//单例工厂方法

#pragma mark - life cycle
//父类的方法, 也就是生命周期

#pragma mark - public method
//公开方法

#pragma mark - private method
//私有方法

#pragma mark - selector
//存放一些selector以供类似button来调用

#pragma mark - data source
//存放数据源

#pragma mark - delegate
//存放代理

#pragma mark - property setter
//属性set方法

#pragma mark - property getter
//属性get方法
